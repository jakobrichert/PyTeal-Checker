This page will act as a blog that focuses on the best practices of PyTeal smart contract development. There may even be a way to incorporate DAO voting for the official best practices of a smart contract. Another idea would be to issue an algorand based token to smart contracts that meet the best practices as voted on by the DAO.

One case of a best practice is outlined in this code from https://pyteal.readthedocs.io/en/stable/examples.html # configure the admin status of the account Txn.accounts[1] # sender must be admin new_admin_status = Btoi(Txn.application_args[1]) set_admin = Seq( [ Assert(And(is_admin, Txn.application_args.length() == Int(2))), App.localPut(Int(1), Bytes("admin"), new_admin_status), Return(Int(1)), ] ) # NOTE: The above set_admin code is carefully constructed. If instead we used the following code: # Seq([ # Assert(Txn.application_args.length() == Int(2)), # App.localPut(Int(1), Bytes("admin"), new_admin_status), # Return(is_admin) # ]) # It would be vulnerable to the following attack: a sender passes in their own address as # Txn.accounts[1], so then the line App.localPut(Int(1), Bytes("admin"), new_admin_status) # changes the sender's admin status, meaning the final Return(is_admin) can return anything the # sender wants. This allows anyone to become an admin!
